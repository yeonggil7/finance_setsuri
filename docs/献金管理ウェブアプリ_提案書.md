# 献金管理ウェブアプリケーション 導入提案書

**作成日：** 2026年2月24日
**作成者：** 財政部

---

## 1. 背景と目的

### 1-1. 現状の課題

現在、全国の教会における月次財政報告は、教団事務局が配布する **Excelマクロ付きファイル（.xlsm）** を各教会の献金管理者が手元のPCで運用しています。この運用には以下の課題があります。

| # | 課題 | 詳細 |
|---|------|------|
| 1 | **属人化** | 各教会で1台のPCにデータが保存されており、担当者不在時に対応できない |
| 2 | **バージョン管理の困難** | 年度ごとにExcelファイルを更新・配布しており、旧版の使用やデータ移行ミスが発生する |
| 3 | **集計作業の負担** | 教団事務局が全教会分のExcelを集約する作業に大きな工数がかかる |
| 4 | **マクロの互換性** | macOS環境やモバイル端末ではマクロが動作しない場合がある |
| 5 | **リアルタイム把握不可** | 教団本部が各教会の財政状況をリアルタイムに把握できない |

### 1-2. 提案の概要

上記課題を解決するため、既存のExcelシートの構造・運用ルールを忠実に再現した **ウェブアプリケーション** を開発・導入することを提案します。

---

## 2. システム概要

### 2-1. 開発済みプロトタイプ

既にプロトタイプ（動作する試作版）を開発済みです。現行Excelの勘定科目体系、計算ロジック、振込情報を完全に再現しています。

### 2-2. 主要機能

| 機能 | 説明 | 現行Excelとの対応 |
|------|------|-------------------|
| **基礎データ設定** | 教会名、年度、宣教会会費割合、前年繰越残高等を設定 | 「基礎データ」シート |
| **月次収支入力** | 収入の部（8カテゴリ）・支出の部（11カテゴリ）の金額と備考を入力 | 「1月分」～「12月分」シート |
| **振込情報管理** | 4つの銀行口座ごとの振込金額を管理 | 各月シート右側の「振込情報」欄 |
| **月締め機能** | 月次報告を確定し、以降の編集をロック | 「今月分のシートを締める」ボタン |
| **ダッシュボード** | 収支グラフ、残高推移、献金内訳、報告状況を一覧表示 | 「収支グラフ」「残高グラフ」「献金グラフ」シート |
| **年間推移表** | 全科目の12ヶ月分金額を一覧表示（年間合計・月平均付き） | 「年間推移」シート |

### 2-3. 操作イメージ

```
┌─────────────────────────────────────────────────────┐
│  献金管理                                            │
│  月次財政報告システム                                  │
│─────────────────────────────────────────────────────│
│                                                     │
│  📊 ダッシュボード    ← 全体の状況をひと目で把握       │
│  📅 月次報告         ← 毎月の収支を入力（メイン機能）  │
│  📈 年間推移         ← 12ヶ月の一覧表                │
│  ⚙ 基礎データ       ← 年度初めに1回設定              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 2-4. 現行Excelからの改善点

| 改善点 | 内容 |
|--------|------|
| **どこからでもアクセス** | PC・スマートフォン・タブレットのブラウザから利用可能 |
| **データの一元管理** | 全教会のデータがクラウド上に集約される |
| **リアルタイム集計** | 教団事務局が各教会の報告状況・財政状況をリアルタイムに確認可能 |
| **操作ガイド内蔵** | 各科目にツールチップで説明が表示される（Excelの「H列」相当） |
| **自動バックアップ** | データベースの自動バックアップにより、データ消失リスクを低減 |
| **バージョン管理不要** | 科目追加・変更はシステム側で一括更新。教会側の作業は不要 |

## 3. AWS インフラ構成（提案）

### 3-1. 構成方針

**コスト最小化** を最優先とし、小規模利用（数十～数百教会）に最適な構成を提案します。

### 3-2. 推奨構成：AWS App Runner + Amazon RDS

```
   ┌──────────────┐
   │  ブラウザ     │  ← 各教会の献金管理者
   │ (PC/スマホ)   │
   └──────┬───────┘
          │ HTTPS
          ▼
   ┌──────────────┐
   │  CloudFront   │  ← CDN（静的ファイル配信高速化）
   │  (任意)       │
   └──────┬───────┘
          │
          ▼
   ┌──────────────┐
   │  App Runner   │  ← アプリケーションサーバー
   │  (Next.js)    │     Dockerコンテナで自動スケール
   │  0.25 vCPU    │     アクセスがないときは自動停止
   │  0.5 GB RAM   │
   └──────┬───────┘
          │
          ▼
   ┌──────────────┐
   │  Amazon RDS   │  ← データベース
   │  (PostgreSQL) │     db.t4g.micro（無料利用枠あり）
   │  20GB SSD     │
   └──────────────┘
```

### 3-3. 各コンポーネントの役割と選定理由

| コンポーネント | 役割 | 選定理由 |
|----------------|------|----------|
| **AWS App Runner** | Next.jsアプリの実行環境 | サーバー管理不要。Dockerイメージをデプロイするだけ。アクセスがない時間帯はコスト削減可能 |
| **Amazon RDS (PostgreSQL)** | データベース | マネージド型で自動バックアップ。db.t4g.microは**AWS無料利用枠**で12ヶ月間無料 |
| **Amazon ECR** | Dockerイメージの保管 | App Runnerへのデプロイ用。500MBまで無料 |
| **CloudFront（任意）** | CDN | 静的ファイルのキャッシュ・配信高速化。月1TB以下は無料枠あり |

### 3-4. 代替構成（さらにコストを抑える場合）

#### 案A：AWS Lightsail（最安構成）

| 項目 | 内容 | 月額 |
|------|------|------|
| Lightsail インスタンス | 0.5GB RAM / 1vCPU / 20GB SSD | **$3.50（約530円）** |
| Lightsail データベース | 0.5GB RAM / 20GB SSD | **$15.00（約2,250円）** |
| **合計** | | **約2,780円/月** |

- 最もシンプルで安価。小規模利用に最適
- サーバー1台構成のため冗長性はなし

#### 案B：App Runner + RDS（推奨構成）

| 項目 | 内容 | 月額（概算） |
|------|------|------|
| App Runner | 0.25vCPU / 0.5GB / 自動一時停止あり | **$5～15（約750～2,250円）** |
| RDS db.t4g.micro | PostgreSQL / 20GB / 自動バックアップ | **無料（初年度）→ $13程度** |
| ECR | イメージ保管 | **無料～$1** |
| **合計（初年度）** | | **約750～2,250円/月** |
| **合計（2年目以降）** | | **約2,700～4,200円/月** |

- 自動スケール・自動復旧あり
- サーバー管理がほぼ不要

#### 案C：EC2 + RDS（従来型構成）

| 項目 | 内容 | 月額（概算） |
|------|------|------|
| EC2 t4g.micro | 1vCPU / 1GB RAM | **無料（初年度）→ $8程度** |
| RDS db.t4g.micro | PostgreSQL / 20GB | **無料（初年度）→ $13程度** |
| **合計（初年度）** | | **ほぼ無料** |
| **合計（2年目以降）** | | **約3,150円/月** |

- AWSの無料利用枠を最大限活用できる（初めてのAWSアカウントの場合）
- サーバーの保守作業（OS更新等）が発生する

### 3-5. 構成比較まとめ

| | 案A Lightsail | 案B App Runner (推奨) | 案C EC2+RDS |
|---|---|---|---|
| **月額コスト** | 約2,780円 | 約750～4,200円 | 無料～3,150円 |
| **初期設定の簡単さ** | ◎ 簡単 | ○ やや簡単 | △ やや複雑 |
| **運用の手間** | ○ 少ない | ◎ ほぼ不要 | △ OS管理あり |
| **自動スケール** | × なし | ◎ 対応 | △ 手動 |
| **自動バックアップ** | ○ DB対応 | ○ DB対応 | ○ DB対応 |
| **向いている規模** | 小規模 | 小～中規模 | 小～中規模 |

## 4. セキュリティ対策

| 項目 | 対策 |
|------|------|
| **通信の暗号化** | HTTPS（SSL/TLS）を標準適用 |
| **認証** | ログイン機能（メールアドレス＋パスワード）を実装予定 |
| **権限管理** | 教会管理者（自教会のみ編集可）/ 教団事務局（全教会閲覧可）の2ロール |
| **データバックアップ** | RDSの自動バックアップ（7日間保持） |
| **アクセス制御** | AWSセキュリティグループによるネットワーク制限 |

## 5. 導入スケジュール（案）

| フェーズ | 期間 | 内容 |
|----------|------|------|
| **Phase 1** 基盤構築 | 2週間 | AWS環境構築、DB移行（SQLite→PostgreSQL）、認証機能追加 |
| **Phase 2** テスト運用 | 1ヶ月 | 数教会でパイロット運用。フィードバック収集・改善 |
| **Phase 3** 全国展開 | 1ヶ月 | 全教会への利用案内・操作説明会の実施 |
| **Phase 4** 安定運用 | 継続 | 運用監視、年度更新対応、機能追加 |

## 6. 今後の拡張可能性

現在のプロトタイプを基盤として、段階的に以下の機能を追加できます。

| 機能 | 説明 | 優先度 |
|------|------|--------|
| **ユーザー認証・権限管理** | 教会ごとのログイン、教団事務局の管理画面 | 高 |
| **複数教会の一括管理** | 教団事務局が全教会の報告状況を一覧確認 | 高 |
| **Excelエクスポート** | 入力データを既存フォーマットのExcelとして出力 | 中 |
| **通知機能** | 締切日のリマインダーメール送信 | 中 |
| **法人収支報告書** | 国・地方公共団体への提出用帳票の自動生成 | 中 |
| **年度繰越** | 年度末の残高を翌年度に自動引き継ぎ | 中 |
| **監査ログ** | 誰がいつ何を変更したかの記録 | 低 |

## 7. コストまとめ

### 初期費用

| 項目 | 費用 |
|------|------|
| アプリケーション開発 | 開発済み（プロトタイプ完成） |
| AWS環境構築 | 社内対応の場合：0円 |
| ドメイン取得（任意） | 約1,500円/年 |

### 月額ランニングコスト

| 構成 | 月額目安 |
|------|----------|
| 最安構成（Lightsail） | **約2,780円/月** |
| 推奨構成（App Runner） | **約750～4,200円/月** |
| 無料枠活用（EC2+RDS、初年度） | **ほぼ0円/月** |

> 参考：現行のExcel運用では、ファイルの配布・回収・集計にかかる人的コスト（事務局スタッフの工数）が発生しています。ウェブアプリ化により、この工数を大幅に削減できます。

## 8. 結論

1. **プロトタイプは開発済み** であり、現行Excelの機能を忠実に再現しています
2. AWS上での運用コストは **月額約750円～4,200円** とリーズナブルです
3. 全国の教会で **ブラウザからいつでもどこでも** 利用可能になります
4. 教団事務局の **集計作業が大幅に効率化** されます
5. 段階的な機能拡張が可能で、将来の要件変化にも対応できます
