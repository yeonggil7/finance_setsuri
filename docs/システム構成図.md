## 4. ネットワーク構成

```
ユーザーのブラウザ
    │
    │  HTTPS (TLS 1.3)
    │  ドメイン: finance-setsuri.vercel.app
    ▼
Vercel Edge Network (グローバルCDN)
    │
    ├─→ 静的アセット: キャッシュから即座に返却
    │
    └─→ API リクエスト (/api/*)
         │
         │  Serverless Function (iad1 リージョン)
         │
         │  PostgreSQL over SSL
         ▼
    Neon PostgreSQL (us-east-1)
         │
         └─ PgBouncer (コネクションプーリング)
              └─ neondb データベース
```

### 通信経路の説明

| 区間 | プロトコル | 暗号化 |
|------|-----------|--------|
| ブラウザ → Vercel | HTTPS (TLS 1.3) | あり（SSL証明書自動発行） |
| Vercel → Neon DB | PostgreSQL over SSL | あり（sslmode=require） |

### URL一覧

| 種別 | URL |
|------|-----|
| 本番サイト | https://finance-setsuri.vercel.app |
| GitHubリポジトリ | https://github.com/yeonggil7/finance_setsuri |
| Vercel管理画面 | https://vercel.com/yeonggils-projects/finance-setsuri |
| Neon管理画面 | Vercel Storage画面からアクセス |

## 6. アプリケーション画面構成

| パス | 画面名 | 機能 |
|------|--------|------|
| `/` | ダッシュボード | 収支サマリー、グラフ表示（棒・折れ線・円グラフ） |
| `/monthly` | 月次報告入力 | 月別の収入・支出・振込の入力・編集 |
| `/annual` | 年間推移 | 12ヶ月分の収支推移表 |
| `/settings` | 基礎データ設定 | 教会名、年度、伝道費率などの基本設定 |

## 7. セキュリティ

| 項目 | 対応状況 |
|------|---------|
| HTTPS通信 | Vercelが自動でSSL証明書を発行・更新 |
| DB接続暗号化 | SSL必須 (sslmode=require) |
| 環境変数管理 | Vercelの暗号化された環境変数ストア |
| DDoS保護 | Vercel Edge Networkに組み込み |
| ソースコード管理 | GitHub（シークレットは.gitignoreで除外） |

## 8. 無料プランの制限

| サービス | 主な制限 |
|---------|---------|
| **Vercel Hobby** | 帯域幅 100GB/月、Serverless実行時間 100時間/月、1プロジェクトあたり1メンバー |
| **Neon Free** | ストレージ 512MB、コンピュート 191時間/月、ブランチ10個 |

> 教会の献金管理用途（数十教会規模）であれば、無料プランで十分に運用可能です。
> 利用規模が拡大した場合は、Vercel Pro ($20/月) + Neon Launch ($19/月) への移行が推奨されます。

## 9. 今後の拡張案

- **認証機能**: NextAuth.js による教会ごとのログイン機能
